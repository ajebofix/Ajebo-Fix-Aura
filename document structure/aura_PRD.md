1. PRODUCT OVERVIEW

Product Name

Aura

Internal Assistant Name

A.J. Rina

Product Positioning

Aura is a private automotive health management platform built by Ajebo Fix.

It is not:
	•	a repair app
	•	a DIY mechanic tool
	•	a fault code generator
	•	a chatbot for random answers

Aura behaves like a:

private automotive clinical system

The experience should feel closer to:
	•	concierge medicine
	•	executive health monitoring
	•	aviation maintenance oversight
than a traditional mechanic platform.

4. USER TYPES

4.1 Owner

Vehicle owner/client.

Permissions
	•	View vehicle health
	•	Chat with Rina
	•	View treatment progress
	•	Request consultations
	•	Access care plans
	•	View records

Restrictions
	•	Cannot alter treatment state
	•	Cannot access admin console
	•	Cannot view internal advisor notes

⸻

4.2 Driver

Assigned operator of vehicle.

Permissions
	•	Report concerns
	•	Chat with Rina
	•	View operational health status
	•	Request escalation

Restrictions
	•	No strategic vehicle control
	•	No financial access
	•	No treatment approvals
	•	No advisor tools

⸻

4.3 Advisor/Admin

Ajebo Fix internal operator.

Permissions
	•	Manage all vehicles
	•	Create consultations
	•	Create assessments
	•	Update treatment plans
	•	Escalate risks
	•	Access physician console
	•	View summaries
	•	Override statuses

⸻

	•	maintain authority

⸻

Conversation Records

	•	user
	•	vehicle
	•	timestamp
	•	concern
	•	escalation level
	•	AI summary
	•	linked consultation
	•	advisor visibility

⸻

“The car made a weird sound while turning.”

	•	severity: monitor
	•	source: user_report

	•	linked vehicle
